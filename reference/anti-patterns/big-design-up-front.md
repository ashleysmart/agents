# Big Design Up Front (BDUF)

Designing the entire system in detail before any code is written, assuming all requirements are known and stable.

**Applies to**: Software Architecture

---

**What it is**: Designing the entire system in detail before any code is
written, assuming all requirements are known and stable.

**Why not**: Requirements change. The design becomes outdated before
implementation is complete. The cost of the design is paid upfront for
value that may never be delivered. Design iteratively — just enough, just
in time.
